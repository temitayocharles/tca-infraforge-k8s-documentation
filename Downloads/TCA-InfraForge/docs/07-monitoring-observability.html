üé® Creating Custom Dashboards

Step-by-Step Dashboard Creation

1. Access Grafana Dashboard Builder

open https://localhost:3000

# Navigate to: Create ‚Üí Dashboard

üö® Alerting & Notification Setup

Creating Effective Alerts

1. Define Alert Conditions

Example: High CPU Usage Alert

# Alert Rule Configuration
alert: HighCPUUsage
expr: cpu_usage_percent > 85
for: 5m
labels:
  severity: warning
annotations:
  summary: "High CPU usage detected"
  description: "CPU usage is above 85% for more than 5 minutes"

2. Configure Alert Routing

# AlertManager Configuration
route:
  group_by: ['alertname', 'severity']
  group_wait: 10s
  group_interval: 10s
  repeat_interval: 1h
  receiver: 'slack-notifications'

receivers:
- name: 'slack-notifications'
  slack_configs:
  - api_url: 'YOUR_SLACK_WEBHOOK_URL'
    channel: '#alerts'
    title: '{{ .GroupLabels.alertname }}'
    text: '{{ .CommonAnnotations.description }}'

3. Set Up Notification Channels

Email Notifications:

receivers:
- name: 'email-notifications'
  email_configs:
  - to: 'team@company.com'
    from: 'alerts@tca-infraforge.com'
    smpt_smarthost: 'smtp.gmail.com:587'

Slack Integration:

receivers:

  - api_url: 'YOUR_SLACK_WEBHOOK_URL'

Alert Best Practices

  • üî¥ Critical: System down, data loss, security breach
  • üü† Warning: Performance degradation, resource exhaustion
  • üü° Info: Notable events, trend changes
  • üîµ Debug: Detailed diagnostic information

Alert Guidelines

  • üìè Thresholds: Set based on historical data, not guesses
  • ‚è±Ô∏è Duration: Use for clause to prevent false alerts
  • üéØ Specificity: Include context (which service, what metric)
  • üîá Silencing: Plan for maintenance windows
  • üìä Aggregation: Group related alerts to prevent storms

üìà Custom Metrics & Business KPIs

Application-Specific Metrics

API Performance Metrics

# Example: Custom metrics in Python Flask app
from prometheus_client import Counter, Histogram, Gauge

# Request counter
REQUEST_COUNT = Counter(
    'http_requests_total',
    'Total HTTP requests',
    ['method', 'endpoint', 'status']
)

# Response time histogram
REQUEST_LATENCY = Histogram(
    'http_request_duration_seconds',
    'HTTP request latency',
    ['method', 'endpoint']
)

# Active connections gauge
ACTIVE_CONNECTIONS = Gauge(
    'active_connections',
    'Number of active connections'
)

# Usage in Flask routes
@app.route('/api/users')
def get_users():
    with REQUEST_LATENCY.labels('GET', '/api/users').time():
        users = get_users_from_db()
        REQUEST_COUNT.labels('GET', '/api/users', '200').inc()
        return jsonify(users)
